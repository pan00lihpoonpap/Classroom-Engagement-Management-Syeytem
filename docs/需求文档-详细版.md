# 基于微信小程序的课堂活跃度管理系统 - 需求文档

## 1. 项目概述

### 1.1 项目背景

#### 1.1.1 教育信息化发展趋势
随着《教育信息化2.0行动计划》的深入推进，高等教育正在经历从传统教学模式向智慧教学模式的转型。课堂教学作为教育的核心环节，其数字化、智能化改革已成为必然趋势。教育部明确提出要推动信息技术与教育教学深度融合，构建网络化、数字化、智能化、个性化、终身化的教育体系。

#### 1.1.2 传统课堂管理存在的问题
当前高校课堂管理普遍面临以下痛点：

**1. 签到方式落后**
- **纸质签到**：耗时长（通常需要5-10分钟），易代签，难以保存和统计
- **点名签到**：大班教学效率低，影响正常教学进度
- **传统考勤机**：需要专用设备，成本高，维护困难

**2. 课堂互动不足**
- 传统举手发言方式参与度低，部分学生不敢或不愿主动参与
- 教师难以快速了解全班学生的理解程度和学习状态
- 缺乏有效的课堂互动工具，教学反馈滞后

**3. 数据统计困难**
- 学生课堂参与数据分散，缺乏系统化记录
- 活跃度评估主观性强，缺乏量化标准
- 数据分析工作量大，难以为教学改进提供及时支持

**4. 过程性评价缺失**
- 传统评价方式过度依赖期末考试，忽视平时表现
- 缺乏对学生课堂参与度的持续跟踪和记录
- 难以实现个性化教学和精准帮扶

#### 1.1.3 移动互联网技术的机遇
微信作为国内最大的社交平台，用户覆盖率超过90%，微信小程序具有以下优势：
- **无需安装**：即用即走，降低使用门槛
- **跨平台支持**：兼容iOS和Android系统
- **开发成本低**：相比原生App开发周期短、成本低
- **用户基数大**：师生均为微信用户，无需额外推广

二维码技术的成熟应用为课堂签到提供了新的解决方案：
- **快速便捷**：扫码即可完成签到，耗时不超过3秒
- **防作弊**：动态二维码+时间限制+地理位置验证，有效防止代签
- **数据准确**：自动记录时间、地点等信息，数据可追溯

#### 1.1.4 项目价值与意义
本系统旨在通过微信小程序和二维码技术，构建一个集签到、互动、统计、分析于一体的智能化课堂管理平台，具有以下价值：

**对教师而言**：
- 节省课堂管理时间，提高教学效率
- 实时掌握学生学习状态，及时调整教学策略
- 获得量化的教学效果数据，支持教学改进
- 为课程考核提供客观的过程性评价依据

**对学生而言**：
- 便捷参与课堂活动，提升学习体验
- 清晰了解自身学习表现，激发学习动力
- 获得个性化的学习反馈和建议

**对学校而言**：
- 提升教学管理信息化水平
- 积累教学大数据，支持教学质量评估
- 推动教学方式创新，提高人才培养质量
- 响应教育信息化政策要求

#### 1.1.5 应用场景
- **大型公共课**：解决百人以上大班签到和互动难题
- **专业课程**：提升小班课堂的互动质量和参与度
- **实验课程**：记录实验过程中的操作和表现
- **讲座活动**：统计参与人数和互动效果
- **在线课堂**：配合线上教学进行远程互动管理

### 1.2 项目目标

#### 1.2.1 总体目标
构建一个基于微信小程序的智能化课堂活跃度管理系统，通过二维码技术和数据分析手段，实现课堂签到、互动、统计、分析的全流程数字化管理，提升课堂教学质量和学生参与度，为教学评价和改进提供数据支撑。

#### 1.2.2 具体目标

**1. 提升签到效率与准确性**
- 将课堂签到时间从传统的5-10分钟缩短至1分钟以内
- 通过动态二维码+多重验证机制，将代签率降低至5%以下
- 实现签到数据的自动记录和实时统计，准确率达到99%以上
- 支持100人以上大型课堂的并发签到需求

**2. 丰富课堂互动形式**
- 提供多样化的互动方式：签到、提问、投票、答题等
- 提高学生课堂参与率，目标达到80%以上
- 实现师生实时互动反馈，响应时间不超过2秒
- 降低学生参与门槛，让内向学生也能积极参与

**3. 实现数据智能统计与分析**
- 自动采集和记录学生课堂参与的全过程数据
- 建立科学的活跃度评价模型，量化学生课堂表现
- 生成多维度的数据分析报告（个人、课程、班级、时间等维度）
- 提供可视化图表展示，直观呈现数据趋势和规律

**4. 支持过程性评价体系**
- 为教师提供客观的平时成绩评定依据
- 记录学生整个学期的课堂参与轨迹
- 支持自定义活跃度计算权重和评分标准
- 导出符合教务系统要求的成绩报表

**5. 优化教学管理流程**
- 简化教师课堂管理操作，减少非教学时间占用
- 提供便捷的课程和学生管理功能
- 实现活动创建、发布、统计的一站式操作
- 支持批量导入学生名单和批量导出数据

**6. 保障系统稳定与安全**
- 系统可用性达到99%以上，支持7×24小时稳定运行
- 确保用户数据安全，通过加密存储和传输保护隐私
- 防范各类作弊行为，维护评价体系的公平性
- 提供完善的日志记录和异常处理机制

#### 1.2.3 预期成果

**技术成果**
- 完成一套功能完整的微信小程序系统（学生端+教师端）
- 开发一套稳定可靠的后端API服务
- 建立规范的数据库设计和接口文档
- 实现动态二维码生成和防作弊算法

**应用成果**
- 系统可在实际教学场景中稳定运行
- 支持多门课程、多个班级同时使用
- 积累真实的教学数据，验证系统有效性
- 获得师生的正面反馈和认可

**创新点**
- 将二维码技术与课堂管理深度结合
- 建立多维度的课堂活跃度评价模型
- 实现课堂互动的实时化和数据化
- 为教育信息化提供可复制的解决方案

### 1.3 用户角色

本系统设计两类用户角色：教师和学生，各自拥有不同的功能权限和操作界面。

#### 1.3.1 教师角色

**基本信息**
- 工号、姓名、所属院系、联系方式
- 个人头像、简介等资料

**核心功能权限**

**1. 活动管理**
- **发起活动**
  - 签到活动：课堂考勤打卡
  - 答题活动：课堂随堂问答
  - 测验活动：课堂小测验（支持选择题自动批改）
  - 投票活动：课堂投票表决
- **活动控制**
  - 开始/暂停/结束活动
  - 生成和展示动态二维码
  - 实时查看参与情况
  - 设置活动参数（时间限制、人数限制等）

**2. 课程管理**
- **课程创建与编辑**
  - 创建新课程（课程名称、代码、时间、地点）
  - 编辑课程信息
  - 设置课程封面和简介
- **课程归档与删除**
  - 归档已结束的课程
  - 删除不需要的课程

**3. 班级管理**
- **班级创建与管理**
  - 在课程下创建班级（如：1班、2班、3班等）
  - 编辑班级信息（班级名称、人数等）
  - 删除或归档班级
  - 查看课程下所有班级列表
- **学生管理**
  - 按班级查看学生列表
  - 批量导入学生名单到指定班级（Excel）
  - 手动添加学生到指定班级
  - 在班级间移动学生
  - 从班级中移除学生
  - 查看学生详细信息
- **班级数据统计**
  - 选择特定班级查看数据
  - 整体出勤率统计（按班级）
  - 班级活跃度分析
  - 各项活动参与率对比
  - 活跃度等级分布
  - 班级内学生排名榜单
  - 多个班级数据对比
- **可视化展示**
  - 单个班级活跃度趋势图（折线图）
  - 各类活动参与对比（柱状图）
  - 活跃度分布情况（饼图）
  - 时间段活跃度热力图
  - 多班级对比图表
- **个人数据查询**
  - 在指定班级中查看单个学生的详细数据
  - 学生历史活动记录
  - 学生各项指标得分
  - 与所在班级平均水平对比
  - 与课程整体平均水平对比

**4. 数据查看与管理**
- **签到管理**
  - 查看签到记录（时间、地点、状态）
  - 补签功能：为特殊情况学生手动补签
  - 修改签到状态（迟到、请假等）
  - 导出签到记录
- **答题信息查看**
  - 查看学生答题内容
  - 查看答题时间和用时
  - 统计答题正确率
  - 查看答案分布情况
- **测验结果管理**
  - 选择题自动批改并生成分数
  - 查看测验成绩分布
  - 查看每个学生的答卷详情
  - 统计题目正确率和区分度
  - 导出测验成绩单
- **统计结果查看**
  - 课程整体数据报告
  - 学期总结报告
  - 活跃度趋势分析
  - 导出数据报表（Excel/PDF）

**5. 个人中心**
- 查看个人资料
- 修改个人信息（姓名、院系、联系方式等）
- 修改头像和简介
- 查看我的课程列表
- 系统设置和偏好配置

#### 1.3.2 学生角色

**基本信息**
- 学号、姓名、班级、专业、联系方式
- 个人头像、简介等资料

**核心功能权限**

**1. 课程参与**
- **加入课程**
  - 通过课程代码加入课程
  - 浏览可选课程列表
  - 查看已选课程
  - 退出课程
- **课程信息查看**
  - 查看课程详情

**2. 活动参与**
- **扫码签到**
  - 扫描二维码完成签到
  - 查看签到状态反馈
  - 查看签到历史记录
- **答题互动**
  - 参与课堂答题活动
  - 提交答案
  - 查看答题结果
- **参与测验**
  - 完成课堂测验
  - 查看测验成绩
  - 查看正确答案和解析
- **参与投票**
  - 参与课堂投票
  - 查看投票结果

**3. 历史记录查看**
- 查看所有活动参与记录
- 查看签到历史
- 查看答题和测验记录
- 查看投票记录

**4. 个人中心**
- 查看个人资料
- 修改个人信息（姓名、班级、专业、联系方式等）
- 修改头像和简介
- 查看我的课程列表
- 系统设置和偏好配置

## 2. 功能需求

### 2.1 用户管理模块

#### 2.1.1 用户注册与登录
- **微信授权登录**
  - 使用微信小程序授权获取用户基本信息
  - 自动绑定微信OpenID
  - 首次登录需完善个人信息（姓名、学号/工号、角色等）

- **角色选择**
  - 学生角色：需填写学号、姓名、班级、专业、联系方式
  - 教师角色：需填写工号、姓名、所属院系、联系方式

#### 2.1.2 个人中心
- **个人信息管理**
  - 查看个人资料
  - 修改个人信息（姓名、院系/班级、联系方式等）
  - 修改头像和简介
  - 查看我的课程列表

#### 2.2.1 课程创建
- **基本信息设置**
  - 课程名称
  - 课程代码（用于学生加入课程）
  - 上课时间
  - 上课地点
  - 课程简介
  - 课程封面图片

#### 2.2.2 课程编辑
- 修改课程基本信息
- 更新课程封面
- 编辑课程简介

#### 2.2.3 课程归档与删除
- 归档已结束的课程
- 删除不需要的课程
- 查看归档课程列表

#### 2.2.4 课程查看（教师）
- 查看我的课程列表
- 查看课程详细信息
- 查看课程下的班级列表

  - 搜索学生

  - 在班级间移动学生

- **数据内容**
  - 学生历史活动记录
  - 学生各项指标得分
  - 与所在班级平均水平对比
  - 与课程整体平均水平对比

#### 2.3.5 班级可视化展示

### 2.4 活动管理模块

#### 2.4.1 活动类型
系统支持四种活动类型：
- **签到活动**：课堂考勤打卡
- **答题活动**：课堂随堂问答
- **测验活动**：课堂小测验（支持选择题自动批改）
- **投票活动**：课堂投票表决

#### 2.4.2 活动创建（教师）
- **选择活动类型**
  - 签到、答题、测验、投票

- **基本参数设置**
  - 活动名称
  - 活动描述
  - 选择目标班级（可多选）
  - 开始时间
  - 结束时间/持续时长
  - 人数限制（可选）

- **答题活动设置**
  - 题目内容
  - 题目类型（单选、多选、简答）
  - 标准答案（可选）
  - 分值设置

- **测验活动设置**
  - 添加多道题目
  - 题目类型（单选、多选）
  - 设置正确答案
  - 设置每题分值
  - 设置总分
  - 是否显示正确答案

- **投票活动设置**
  - 投票主题
  - 投票选项（2-10个）
  - 是否允许多选
  - 是否匿名投票
  - 是否实时显示结果

#### 2.4.3 二维码生成与展示（教师）
- **动态二维码生成**
  - 活动创建后自动生成唯一二维码
  - 二维码包含活动ID和时间戳
  - 支持设置二维码刷新频率（防止代签）
  - 二维码定时自动刷新（可设置30秒/1分钟/2分钟）

- **二维码展示**
  - 大屏幕投影展示
  - 支持全屏显示
  - 显示剩余时间倒计时
  - 显示当前参与人数/总人数
  - 显示参与率百分比

- **二维码安全机制**
  - 时间限制：超时自动失效
  - 地理位置限制（可选）：限定扫码范围
  - 设备限制：同一设备限制扫码次数
  - 动态刷新：定时更新二维码内容

#### 2.4.4 活动控制（教师）
- **活动状态管理**
  - 开始活动
  - 暂停活动
  - 继续活动
  - 结束活动

- **实时监控**
  - 查看实时参与情况
  - 查看参与学生列表
  - 查看未参与学生名单
  - 实时统计参与率

#### 2.4.5 活动参与（学生）
- **扫码参与**
  - 扫描二维码参与活动
  - 实时反馈参与状态（成功/失败/已参与）
  - 查看活动详情

- **签到活动**
  - 扫码完成签到
  - 查看签到状态
  - 查看签到时间

- **答题活动**
  - 查看题目内容
  - 提交答案（文字/选项）
  - 查看提交状态
  - 查看答题结果（如果教师公布）

- **测验活动**
  - 查看测验题目
  - 逐题作答
  - 提交答卷
  - 查看测验成绩（选择题自动批改）

## 6. 接口设计规范

### 6.1 RESTful API设计
- 使用标准HTTP方法（GET、POST、PUT、DELETE）
- 统一的URL命名规范
- 统一的响应格式

### 6.2 响应格式示例
```json
{
  "code": 200,
  "message": "success",
  "data": {
    // 业务数据
  },
  "timestamp": 1635123456789
}
```

### 6.3 错误码定义
- 200：成功
- 400：请求参数错误
- 401：未授权
- 403：禁止访问
- 404：资源不存在
- 500：服务器内部错误
